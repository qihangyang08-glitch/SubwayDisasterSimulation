# 模块四 UI 布局与控件设计规范（V1.0）

## 1. 设计目标
- 服务三类核心任务：初始化配置、历史回放复盘、LLM 预案生成。
- 与 DataStruction V1.1 一致：字段不重命名，交互仅做可视化封装。
- 风格基调：灰、白、深蓝，轻量现代，专业简洁，少圆角。

## 2. 全局布局骨架
- 页面容器：`12 栅格 + 固定顶栏 + 左侧导航 + 右侧可收起信息抽屉`。
- 顶栏高度：56px。
- 左导航宽度：232px。
- 右抽屉宽度：320px（默认折叠）。
- 主区内边距：20px。

## 3. 三个子页面

### 3.1 初始配置（InitConfig Studio）
- 结构：`25% 参数区 | 50% 灾害配置区 | 25% 特殊实体时间线`。
- 区块与控件：
  - 基础参数块：scenarioId/mapLevel/seed/totalPeople。
  - 水灾块：inlets 表格，支持新增/删除/复制。
  - 火灾块：sources 表格，支持 smoke/gas 参数联动校验。
  - 特殊实体块：卡片列表 + 时间轴条，`triggerAt` 必填。
- 关键按钮：
  - 主：创建仿真。
  - 次：校验配置、保存草稿、导入、导出。
  - 危险：重置（弹确认）。

### 3.2 历史查看（Replay & Inspect）
- 结构：`上 70% 画面与指标 | 下 30% 时间轴与图表`。
- 区块与控件：
  - 左上：Pixel Streaming 视频。
  - 右上：KPI 卡片（疏散、伤亡、暴露）。
  - 下方：播放控制条 + 事件轨 + ECharts。
- 关键按钮：
  - 主：订阅仿真、恢复实时。
  - 次：播放/暂停、步进、倍速、上一关键帧、下一关键帧。
- 时间行为：
  - seek 节流 100ms。
  - 以 SimState.currentTime 为主时钟，UpdateFrame.simTime 用于校准。

### 3.3 大模型预案生成（LLM Plan Workbench）
- 结构：`35% 上下文 | 40% Prompt 编辑 | 25% 预览与编译结果`。
- 区块与控件：
  - 上下文区：fromSimulationId/fromSimTime/目标指标。
  - 编辑区：模板、约束、变量填充。
  - 结果区：actions 表格、去重结果、编译计数。
- 关键按钮：
  - 主：生成预案、应用预案。
  - 次：校验时序、一键对齐 fromSimTime、回填到 InitConfig。

## 4. 按钮与表单规范
- 主按钮：深蓝底白字，边角 3px，高度 36px，最小宽度 108px。
- 次按钮：白底深蓝描边。
- 危险按钮：白底红色描边，仅用于清空/覆盖。
- 输入框：高度 34px，边框 1px，聚焦时深蓝外发光 1px。
- 表格行高：36px，表头加粗。

## 5. 设计令牌
- `--color-primary: #163A5F`
- `--color-bg: #FFFFFF`
- `--color-surface: #F7F9FC`
- `--color-border: #E7EBF0`
- `--color-text: #2E3440`
- `--radius-sm: 3px`
- `--radius-md: 4px`
- `--shadow-soft: 0 1px 6px rgba(20, 34, 56, 0.06)`

## 6. 交互动效
- 页面切换：淡入 180ms。
- 列表加载：逐项淡入 120ms，间隔 25ms。
- 禁止高频动画与大幅位移。

## 7. 可用性与验收检查
- 三页面均可用键盘完成核心路径（Tab/Enter/Escape）。
- 任意失败请求均显示 requestId 与可重试按钮。
- 关键动作（创建仿真、应用预案）均有加载态与幂等保护。
- 深浅对比满足可读性要求（文本对比建议不低于 4.5:1）。
