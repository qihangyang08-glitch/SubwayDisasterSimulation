# 操作指南：项目结构与 Pixel Streaming 前端模板

本文档用于：
1) 建立前端与中台的项目目录结构。
2) 在 UE4.26 中启用 Pixel Streaming 并打包。
3) 获取 Pixel Streaming 前端模板（HTML/CSS/JS）并放入前端项目中改造。

---

## 一、项目目录结构（建议）

在项目根目录下创建以下结构（可按需调整命名，但建议固定，方便协作）：

```
/FRONT_UE
  /frontend               前端 Web 项目（React/Vue）
  /pixel_streaming_tpl    Pixel Streaming 前端模板（原始 HTML/CSS/JS）

/MIDDLEWARE
  /server                 中台服务（HTTP + Socket.IO + DB）
  /db                      数据库脚本与迁移文件

/UE_PROJECT
  /MyProject              UE4.26 工程

/DOCS
  /README.md              对接说明与部署说明
```

创建步骤：
1) 新建目录：FRONT_UE、MIDDLEWARE、UE_PROJECT、DOCS。
2) 在 FRONT_UE 下创建 frontend 与 pixel_streaming_tpl。
3) 在 MIDDLEWARE 下创建 server 与 db。

---

## 二、UE4.26 启用 Pixel Streaming 与打包

### 2.1 启用插件
1) 打开 UE4.26 工程。
2) 菜单：Edit -> Plugins。
3) 搜索 Pixel Streaming，勾选启用后重启编辑器。

### 2.2 基础设置（建议）
1) 打开 Project Settings。
2) 在 Plugins -> Pixel Streaming 中确认已启用相关选项。
3) 如需无窗口渲染，后续运行时添加启动参数。

### 2.3 打包
1) 菜单：File -> Package Project -> Windows (64-bit)。
2) 选择输出目录并等待打包完成。

### 2.4 运行时参数（本地测试示例）
启动打包后的 exe 时，建议附带以下参数：

```
-PixelStreamingIP=127.0.0.1 -PixelStreamingPort=8888 -RenderOffScreen -AudioMixer
```

说明：
- PixelStreamingIP/Port 需与信令服务器一致。
- RenderOffScreen 适合后台渲染。

## 三、获取 Pixel Streaming 前端模板（HTML/CSS/JS）

注意：Pixel Streaming 的前端模板不是从“打包产物”中生成的，而是随 UE4.26 的 SignallingWebServer 提供。你需要从引擎目录中拷贝模板文件。

### 3.1 模板所在位置（UE4.26 引擎目录）
引擎目录下通常包含：

```
Engine/Source/Programs/PixelStreaming/WebServers/SignallingWebServer/public
```

该 public 目录内包含：
- HTML 页面文件
- CSS 样式文件
- JS 逻辑文件
- 依赖资源（图片、字体等）

### 3.2 拷贝到前端项目
将 public 目录内的文件完整拷贝到：

```
/FRONT_UE/pixel_streaming_tpl
```

此目录作为“原始模板保留区”，后续可复制一份到前端项目进行改造，避免覆盖原始模板。

---

## 四、前端项目如何使用模板

建议使用两份模板：
- 原始模板：保存在 pixel_streaming_tpl（只读）。
- 改造模板：放在 frontend 内作为页面基础。

### 4.1 放置位置建议
在前端项目中放置模板文件：

```
/FRONT_UE/frontend/public/pixelstreaming
```

好处：
- 打包后可直接作为静态资源访问。
- 便于与现有路由页面共存。

### 4.2 页面改造方式（推荐）
1) 保持播放器核心 JS 逻辑不动。
2) 将 UI 控件与业务面板部分替换为你的页面组件。
3) 仅调整 DOM 结构与样式，避免破坏信令与 WebRTC 初始化逻辑。

---

## 五、放置与部署约定（建议）

1) pixel_streaming_tpl：保留原始模板，禁止直接改动。
2) frontend/public/pixelstreaming：作为前端定制模板入口。
3) 中台与信令服务器部署时，如果使用 UE 自带 SignallingWebServer：
   - 直接替换其 public 目录为前端改造版即可。
4) 如果使用独立前端服务（Nginx/Node 静态服务）：
   - 将 frontend 构建产物部署到静态服务器，由该服务器提供页面。

---

## 六、最小联通检查

1) 启动 SignallingWebServer。
2) 启动打包后的 UE 程序（带 Pixel Streaming 参数）。
3) 浏览器访问前端页面：
   - 能看到播放器画面。
   - 控制输入（鼠标/键盘）能反馈到 UE。
4) 确认端口与 IP 对齐。
