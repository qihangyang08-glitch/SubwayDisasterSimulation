# 🚀 项目名称：数字孪生地铁灾害应急仿真系统
# 📅 阶段：寒假攻坚（MVP原型开发）
# 🎯 目标：产出可联调、可演示的系统雏形

---

## 🛠️ 第一部分：基础设施与规范（全员必读 - Tech Lead 负责）

这部分不产出具体功能，但决定了能不能合到一起。**这是你作为组长/核心要掌控的。**

*   [ ] **T01. 环境统一**
    *   UE版本：统一使用 **UE 5.x**（建议5.3，稳定性好）。
    *   代码仓库：建立 Git 仓库（Gitee/GitHub），设 `Main`（稳定版）和 `Dev`（开发版）分支。
    *   Python环境：统一 Python 3.10+，规定依赖库版本（`requirements.txt`）。
*   [ ] **T02. 接口契约签署（API Definition）**
    *   任务：编写《前后端交互协议文档_v1.0.md》。
    *   核心产出：明确 UE 发送什么（如 `{"event": "fire", "loc": "A"}`），Python 返回什么（如 `{"action": "open_door"}`）。
    *   *负责人：你（Tech Lead）*

## 🎨 第二部分：三维场景与动态表现组（前端/美术）

**核心逻辑：** 静态资产导入 -> 蓝图控制动态。

*   [ ] **T03. 地铁站灰盒搭建 (Greyboxing)**
    *   任务：不贴材质，仅用 UE 自带的几何体（BSP Brush）搭建出站厅、站台、楼梯。
    *   要求：尺寸严格按照《概要设计》执行（如站台长55m）。
    *   *交付物：一个“白模”关卡文件（.umap）。*
*   [ ] **T04. 资产搜集与处理 (Asset Pipeline)**
    *   任务：从 Sketchfab/UE商城 寻找免费模型（闸机、灭火器、NPC、列车壳子）。
    *   工具：Blender（用于转换格式为FBX，修正坐标轴）。
    *   *交付物：`Content/Meshes` 文件夹下的所有静态网格体。*
*   [ ] **T05. NPC 基础行为 (NavMesh)**
    *   任务：
        1. 在地面铺设 `NavMeshBoundsVolume`。
        2. 创建 NPC Character，添加 `DetourCrowdAIController`（防碰撞）。
        3. 编写简单蓝图：游戏开始 -> 随机在场景里走动。
    *   *交付物：一个会在场景里乱跑但不撞墙的 NPC 蓝图。*
*   [ ] **T06. 灾害特效实装 (Niagara)**
    *   任务：从 StarterContent 提取 Fire/Smoke 粒子，制作成可生成的 Actor。
    *   *交付物：`BP_Fire`，拖进场景就能看着火的蓝图类。*

## ⚡ 第三部分：仿真算法组（C++ / 蓝图逻辑）

**核心逻辑：** 寻找开源算法 -> C++封装 -> 暴露给蓝图。

*   [ ] **T07. 算法选型与验证 (Research)**
    *   任务：在 GitHub 搜索 `Cellular Automata Smoke CPP` 或 `Grid based fluid simulation`。
    *   目标：找到一个轻量级的 C++ 类，能输入格子坐标，算出周围格子的密度。
    *   *交付物：选定的 GitHub 链接和本地跑通的 Demo。*
*   [ ] **T08. C++ 算法封装 (The Wrapper)**
    *   任务：
        1. 创建 UE C++ 类，继承 `UBlueprintFunctionLibrary`。
        2. 将 T07 的算法逻辑复制进去。
        3. 使用 `UFUNCTION(BlueprintCallable)` 宏暴露函数。
    *   *交付物：一个名为 `USimulationHelper` 的 C++ 类。*
*   [ ] **T09. 蓝图逻辑串联 (Logic Binding)**
    *   任务：在蓝图中调用 T08 写的节点。
    *   逻辑：每秒调用一次算法 -> 获取有烟雾的坐标 -> 在这些坐标处生成/移动粒子特效。
    *   *交付物：`BP_SimulationManager`，放入场景即可控制烟雾扩散。*

## 🧠 第四部分：大模型决策组（后端 / AI）

**核心逻辑：** 知识库构建 (RAG) -> Prompt 约束 -> 接口服务。

*   [ ] **T10. 知识库构建 (Knowledge Base)**
    *   任务：编写/整理 `subway_emergency_rule.txt`。
    *   内容：包含“火灾处置流程”、“洪水处置流程”、“疏散广播话术模板”。
    *   *交付物：清洗好的纯文本数据文件。*
*   [ ] **T11. 提示词工程 (Prompt Engineering)**
    *   任务：在 Playground（如 ChatGPT 网页版）调试 Prompt。
    *   **关键要求：**
        1.  System Prompt：设定为地铁专家。
        2.  Input：插入 T10 的知识库内容 + 灾害状态。
        3.  Output Format：**强制要求 JSON 格式**（定义好Schema）。
    *   *交付物：一段测试通过的 Prompt 模板字符串。*
*   [ ] **T12. 后端服务开发 (FastAPI)**
    *   任务：
        1. 写一个 `server.py`，开放 `/api/decision` 接口。
        2. 接入大模型 API（通义千问/讯飞星火）。
        3. 接收请求 -> 组装 Prompt -> 调 API -> 返回 JSON。
    *   *交付物：本地运行 `python server.py` 后，用 Postman 能调通接口。*

*   [ ] **T13. UE HTTP 通信测试**

    *   *交付物：寒假最终演示视频。*
