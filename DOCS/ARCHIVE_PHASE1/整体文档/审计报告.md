**文件：** `src/socket/dispatcher.js` (591行)

**状态机设计：**

  [paused]
     |

**时间推进公式：**
```javascript
// 每个tick执行（默认1000ms）
t_next = t_current + (tickMs / 1000) * playbackSpeed

// 示例：2x倍速时
// t_current = 10.0s, tickMs = 1000ms, playbackSpeed = 2.0
// t_next = 10.0 + 1.0 * 2.0 = 12.0s
```

**✅ 优点：**
- 播放/暂停/跳转/变速控制完整
- 帧乱序过滤机制（按simTime + frameIndex排序）
- 断线重连自动补发当前帧

**❌ 缺陷：**

**🔴 BUG-002：Socket断线时session未清理**

**问题代码：**
```javascript
// dispatcher.js - 缺少disconnect事件处理
io.on('connection', (socket) => {
  // ... 各种事件监听

  // ❌ 缺少以下代码：
  // socket.on('disconnect', () => {
  //   sessions.delete(socket.id);  // 应该清理session
  // });
});
```

**影响：** 
- 每个连接创建一个session对象（包含定时器、缓存等）
- 断开连接后session永不删除
- 长期运行会导致内存持续增长，最终OOM

**修复方案：**
```javascript
io.on('connection', (socket) => {
  const sessionId = socket.id;
  
  // ... 现有事件监听
  
  // ✅ 添加清理逻辑
  socket.on('disconnect', () => {
    const session = sessions.get(sessionId);
    if (session) {
      // 清理定时器
      if (session.timer) clearInterval(session.timer);
      // 删除session
      sessions.delete(sessionId);
      console.log(`[Dispatcher] Session ${sessionId} cleaned up`);
    }
  });
});
```

**优先级：** 🔴 P0 - 立即修复

  lastTickTime = now;
  
  session.currentTime += actualDelta * playbackSpeed;  // ✅ 使用实际时间
  // ...
