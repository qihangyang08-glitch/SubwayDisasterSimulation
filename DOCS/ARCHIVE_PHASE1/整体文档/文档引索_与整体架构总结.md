# 📚 数字孪生地铁灾害应急仿真系统 - 文档与模块导读

由于本项目包含大量的需求文档、设计文档以及不同阶段的交付报告，特此提供此全局文档引索，方便快速定位相关内容并总览项目当前状态。

## 🏗️ 整体测试与运行说明 (当前阶段: 前后端+中台联调阶段)
根据当前进度，项目前三大模块（中台核心、数据库、Socket通信）已开发完成，**前端及信令服务器 (Pixel Streaming)** 也已完成核心交互绑定。

### 🚀 一键全启及测试流程
1. 运行 `PROJECT/start-all.ps1` 脚本（以管理员身份运行效果更佳）。该脚本将依次：
   - 检查并启动 **MySQL 8** 数据库。
   - 启动 **Node.js 中台服务**（依赖 `.env` 中的数据库配置，监听 3000 端口）。
   - 启动基于 Epic 官方改造的 **信令服务器 (SignallingWebServer)** （运行于 80 端口）。
   - 自动拉起打包好的 **UE 可执行程序**。
2. 浏览器打开 `http://localhost/player.html` （无需等待，页面加载1秒后会自动建立以下所有连接：）
   - **自动连接 Socket.io** -> 到 Node.js 中台。
   - **自动建立 WebRTC 像素流** -> 到信令服务器及 UE 进程。
3. **数据库验证**
   - 点击前端界面的 **[创建仿真]** 按钮。
   - 观察控制台日志显示“仿真创建成功”。
   - 打开 Navicat 连接本地 MySQL (库名 `metro_simulation`)，查看 `simulations` 表，即可看到刚才页面触发创建的最新一条数据。

## 📂 各层文档大全与分布

### 1. 根目录：需求与系统设计阶段档案
这部分文件是在项目开发初期，由于业务拆解和提示词工程生成的设计级文档。
- `任务清单.md` / `前端任务清单.md`：记录了各个子模块需要完成的具体任务。
- `DataStruction.md`：详细约束了数据库表结构与各模块的数据传输规范。
- `操作指南_项目搭建.md`：早期的环境配置说明。
- `详细设计_XX.txt` / `第一次会议总结.txt`：记录了早期的沟通思路与技术栈选型。
- `理解.md`：业务逻辑流程的梳理，包括灾害仿真的物理计算机制理解。

### 2. PROJECT/ 目录：工程与测试统筹档案
这是进入实际代码执行阶所生成的全局工程文档。
- `README.md` & `README_UPDATES.md`：项目的核心介绍，及最近几次快速修复和补丁的更新记录。
- `E2E_TEST_GUIDE.md` & `TEST_GUIDE.md`：端到端（UE -> 中台 -> 前端）整体联调的方法论及测试用例。
- `COMPLETION_REPORT.md`：各阶段开发的完成度报告与总结。

### 3. PROJECT/MIDDLEWARE/ 目录：中台模块交付档案
中台的核心架构（API路由、Socket引擎、DB持久化等）。
- `MIDDLEWARE/README.md`：有关中台开发与文件结构的入门介绍。
- `MIDDLEWARE/db/` 类档案（如 `IMPLEMENTATION_SUMMARY.md`, `TECHNICAL_GUIDE.md`, `CHANGELOG.md`）：全面记录了数据库 Schema 的生成规则、事务锁的考虑和当前的数据结构快照。
- `MIDDLEWARE/server/docs/`：**包含 Module 1 到 Module 5 核心执行的 Runbook**，记录了每一次代码生成的审计矩阵、测试覆盖报告和校验清单（非常丰富的历史执行轨迹日志）。

### 4. PROJECT/FRONT_UE/ 目录：前端及引擎集成档案
- `FRONT_UE/README.md`：简述UE前端结合的意图。
- `FRONT_UE/frontend/docs/` (包含 `module4_execution_plan...` 和 `module4_ui_layout_spec.md`)：
  这些是前端网页播放器（信令及交互控制UI）开发的原始蓝导图及界面排版规范（UI / UX 细分方案）。
- **`FRONT_UE/frontend/SignallingWebServer/player.html`**：项目当前最终产出的入口页，内置控制面板与像素流连接逻辑。
