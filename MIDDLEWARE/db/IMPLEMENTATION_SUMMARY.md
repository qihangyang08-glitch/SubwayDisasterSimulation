# 数据库自检与初始化功能 - 实施总结

## 📋 任务概述

为"数字孪生地铁灾害应急仿真"项目添加**数据库自检与初始化**功能，实现系统启动时自动检查数据库状态，若数据库或表不存在则自动创建和初始化。

## ✅ 完成的工作

### 1. 核心模块开发

#### 1.1 dbInitializer.js (核心初始化模块)
**位置**: `src/db/dbInitializer.js`

**功能**:
- ✅ 检查数据库是否存在
- ✅ 自动创建数据库（如不存在）
- ✅ 检查所有必需的表是否存在
- ✅ 自动创建缺失的表
- ✅ 验证表结构完整性
- ✅ 详细的日志输出

**主要API**:
```javascript
initializeDatabase(options)      // 完整初始化流程
checkConnection(options)          // 连接测试
checkDatabaseExists(conn, dbName) // 检查数据库
checkTableExists(conn, db, table) // 检查表
checkAllTables(conn, dbName)      // 检查所有表
validateTableStructure(conn, db)  // 验证表结构
```

## 📊 数据库表结构

系统自动创建以下4张表:

1. **simulations** - 仿真配置信息
2. **simulation_frames** - 仿真帧数据
3. **plans** - 应急计划
4. **plan_runs** - 计划执行记录

详细结构见文档: `db/README_DB_INIT.md`


## 📝 配置步骤

1. **复制配置文件**
   ```bash
   cp .env.example .env
   ```

2. **修改数据库配置**
   ```bash
   # 编辑 .env 文件
   DB_HOST=localhost
   DB_PORT=3306
   DB_USER=root
   DB_PASSWORD=your_password
   DB_NAME=metro_sim
   ```

3. **启动服务**
   ```bash
   npm start
   ```
   或先初始化数据库:
   ```bash
   npm run db:init
   npm start
   ```

## ⚠️ 注意事项

1. **权限要求**: 数据库用户需要有创建数据库和表的权限
2. **MySQL版本**: 建议使用MySQL 8.0+
3. **备份数据**: 在生产环境操作前务必备份数据
4. **配置检查**: 确保.env文件配置正确

## 🔍 故障排查

### 无法连接MySQL
```bash
# 检查MySQL服务是否运行
# Windows:
net start MySQL80

# Linux/Mac:
systemctl status mysql
```

### 权限不足
```sql
-- 在MySQL中授予权限
GRANT ALL PRIVILEGES ON *.* TO 'root'@'localhost';
FLUSH PRIVILEGES;
```

### 表结构验证失败
- 检查是否手动修改过表结构
- 考虑重新创建表（备份数据后）
