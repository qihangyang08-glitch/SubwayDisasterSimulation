# 事项说明与决策记录

更新时间：2026-04-01

## 1. 当前共识

1. 系统由 UE 客户端、前端信令服务、中台服务三部分组成。
2. 在环境正确配置下，可按 VISUAL_TEST_RUNBOOK 完成联调。
3. 前端第二轮功能已完成主要交互：创建仿真细化、历史页拆分、分析占位。

## 2. 已确认问题

1. 运行手册中 UE 快捷方式启动命令曾有写法错误（已修正为带引号的 Start-Process 调用）。
2. 历史数据分析功能目前仅有交互占位，尚无真实绘图和分析计算。
3. 是否完成全链路落库，仍需通过可执行用例逐项验证并保留证据。

## 3. 技术决策

### 3.1 前端与中台保留 HTTP + Socket 双连接

- HTTP 用于创建、查询、配置拉取等事务型接口。
- Socket 用于实时状态、进度和异步事件推送。
- 两者并存是当前阶段的稳定设计，不建议删除其一。

### 3.2 特殊实体固定位配置归属

建议职责如下：

1. UE 场景设计者定义合法位置、容量、类型原始集合。
2. 中台维护配置中心并对外提供查询接口，负责最终校验。
3. 前端初始化拉取后渲染，不硬编码业务点位规则。

## 4. 下一阶段执行建议（你提出的方案落地化）

### 目标

构建一个“模拟仿真后端”，用于验证跨模块通信和数据库链路，而不依赖真实算法。

### 执行边界

1. 前端提交新建仿真给中台。
2. 中台创建仿真实体并通知模拟后端开始产帧。
3. 模拟后端按 DataStruction 严格格式返回一批帧。
4. 中台完成落库并向前端发送完成事件。
5. 测试者通过数据库与历史页验证链路。

详细任务拆分见：DOCS/PROCESS/NEXT_PHASE_EXECUTION_PLAN.md

## 5. 测试证据要求

建议每轮联调至少留存以下证据：

1. 启动日志截图（中台、信令、UE）。
2. 健康检查与端口检测结果。
3. 前端关键操作录屏（创建仿真、回放、历史查看）。
4. 数据库查询结果（simulations 与 simulation_frames）。
5. 异常日志与处理结论。
